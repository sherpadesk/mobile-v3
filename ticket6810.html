    I found interesting idea on (https://habrahabr.ru/company/microsoft/blog/327258/)
    its only in russian, so I describe it.

    the idea is use one Bot instance to gather all conversations from all channels (slack, skype, email, web) to one Bot

    then bot can parse input and respond to one of available client channels (slack, skype, email, web).

    So we even dont need chat messenger for tickets.
    Clients can use their own favorite client (ie Skype). Then connect(auth) it to our bot.
    Tech use context of ticket to post response. Then bot add post to Sherpadesk as ticket post and system send it to user on available channel (email, skype, slack). So user see message from tech and respond in his client (ie email).

    So we have live conversation with log stored in Sherpadesk. So user and tech can see full history in ticket.

    Example:

    Bot> Hello!

    Tech> ticket #12

    Bot> last reply on opened ticket by Mike "is this done?"

    Tech> "tell Mike or just tell" - "in process now"
    or close ticket with message: "yes."

    Bot>your response saved to ticket. Email sent to Mike ... (all users of this thicket)
    (added response, and sent notification to Mike last channel (ie Skype, sms) and email)

    Bot>what do you want to do next?
    Bot>Mike is online now...
    Bot>new message received from Mike: "argh, I need it ASAP!!!"

    Tech>call Mike with skype (or send sms) or slack.

    So all conversation will be saved in Ticket #12
